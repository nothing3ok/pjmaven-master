本版本中完成响应客户端404 页面的工作

当客户端请求的资源没有找到时候，我们应当响应客户端404页面

404页面是一个公共页面，不属于某个具体的webapp 中的资源时，只要不存在都应当
响应这个页面

实现
1 在webapps目录中建立一个子目录：root
2 在root文件夹里面建立404.html
3 由于影响404页面的同时还要将当时响应中状态行中代码与描述改为404  NOT FOUND
因此我们要将HTTP response中发送状态行的操作进行修改，不能发送固定的200 ok 应当改为可以进行设置的
 3.1 在httpresponse 中台南佳两个新的属性
 int statuscode=200    状态代码
 string status reason=”ok“  状态描述
 并且赋予他们默认值200，ok 因为大部分的请求我们都应当正确处理，所以正确的就无需要在设置两项

 3.2 提供这两个属性的 get，set方法，共外界可以进行设置
 3.3 修改sendstausline 方法，将状态行内容中状态代码与描述部分改为使用这个两个属性进行发送
 4 在clientgandler的处理请求分支中，当资源没有找到时，设置response 应当404 页面 ，同时
 设置好对应的状态代码与状态描述 404 not found
